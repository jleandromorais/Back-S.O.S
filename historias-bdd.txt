## HU 01 - Denúncia Anônima
**História de Usuário**  
Como aluno,  
Eu gostaria de realizar uma denúncia anônima,  
Para que eu possa relatar casos de bullying sem medo de represálias.  

**Cenário 1 (positivo) – Denúncia registrada com sucesso**  
- Dado que estou logado como aluno  
- E selecionei a opção de denúncia anônima  
- Quando preencho os campos obrigatórios (tipo de violência, descrição, data/local)  
- Então o sistema deve registrar a denúncia anonimamente  
- E não deve armazenar nenhum dado pessoal meu  

**Cenário 2 (negativo) – Denúncia sem campos obrigatórios**  
- Dado que estou logado como aluno  
- Quando tento enviar uma denúncia sem preencher todos os campos obrigatórios  
- Então o sistema deve exibir a mensagem “Por favor, preencha todos os campos obrigatórios”  
- E não deve permitir o envio até a correção

# HU 02 - Análise de Denúncias
## História de Usuário
**Como** um gestor (coordenador ou diretor)  
**Eu quero** visualizar as denúncias anônimas  
**Para que** eu possa analisá-las, identificar padrões e tomar as devidas providências

## Cenários

**Cenário 1 (positivo) – Visualização de denúncias registradas**  
- Dado que estou logado como gestor  
- E existe pelo menos uma denúncia registrada no sistema  
- Quando acesso a área de "Denúncias Recentes"  
- Então o sistema deve exibir uma lista com as denúncias  
- E cada denúncia deve mostrar o tipo, a descrição, a data e o status de anonimato  

**Cenário 2 (negativo) – Nenhuma denúncia registrada**  
- Dado que estou logado como gestor  
- E não há nenhuma denúncia registrada no sistema  
- Quando acesso a área de "Denúncias Recentes"  
- Então o sistema deve exibir uma mensagem informando que não há denúncias a serem exibidas

# HU 03 - Protocolo de Acompanhamento
## História de Usuário
**Como** aluno  
**Eu quero** receber um protocolo automático para cada denúncia  
**Para que** eu possa rastrear o andamento da denúncia

**PO:** O protocolo é gerado automaticamente após realizar a denúncia para fazer o acompanhamento.

## Cenários

**Cenário 1 (positivo) – Protocolo gerado com sucesso**  
- Dado que estou logado como aluno  
- E preenchi todos os campos obrigatórios para realizar a denúncia  
- Quando envio a denúncia  
- Então o sistema deve gerar um número único de protocolo  
- E exibir o protocolo na tela para acompanhamento  
- E permitir que eu acompanhe o status da denúncia

**Cenário 2 (negativo) – Falha ao gerar protocolo**  
- Dado que estou logado como aluno  
- E ocorre um erro no envio da denúncia  
- Quando tento visualizar o protocolo  
- Então o sistema deve exibir uma mensagem de erro  
- E não deve gerar um protocolo até que a denúncia seja enviada com sucesso

# HU 04 - Login com Autenticação para Gestores

## História de Usuário
**Como** psicólogo escolar  
**Eu quero** acessar o sistema com login seguro  
**Para que** eu possa visualizar denúncias garantindo a confidencialidade dos dados  

**PO:** Implementar uma autenticação simples para o login

## Cenários

### Cenário 1 (positivo) – Login bem-sucedido
**Dado que** sou um gestor cadastrado no sistema  
**Quando** insiro meu email e senha corretos  
**Então** o sistema deve me autenticar  
**E** redirecionar para o painel administrativo  

### Cenário 2 (negativo) – Credenciais inválidas
**Dado que** sou um gestor cadastrado no sistema  
**Quando** insiro credenciais incorretas  
**Então** o sistema deve exibir mensagem de erro

# HU 05 - Feedback do Denunciante

## História de Usuário
**Como** aluno  
**Eu quero** avaliar o atendimento da escola após a denúncia  
**Para que** eu possa saber se minha voz foi ouvida  

**PO:** Alterar número de SAC para cada escola.

## Cenários

### Cenário 1 (positivo) – Feedback enviado com sucesso
**Dado que** fiz uma denúncia há mais de 7 dias  
**Quando** acesso o sistema para avaliar o atendimento  
**Então** devo receber um formulário de satisfação anônimo  
**E** poder enviar meu feedback sobre o processo  

### Cenário 2 (negativo) – Feedback fora do prazo
**Dado que** fiz uma denúncia há menos de 7 dias  
**Quando** tento acessar o formulário de feedback  
**Então** o sistema deve informar que a avaliação estará disponível após 7 dias da denúncia

# HU 06 - Mensagens de Encerramento de Caso

## História de Usuário
**Como** denunciante  
**Eu quero** receber uma mensagem ao final da análise da denúncia  
**Para que** eu possa saber que ela foi finalizada  

**PO:** Colocar um alerta que dá o trigger quando a classificação da denúncia altera para "Finalizado"

## Cenários

### Cenário 1 (positivo) – Mensagem de encerramento enviada
**Dado que** minha denúncia foi classificada como "Finalizado"  
**Quando** o sistema atualiza o status  
**Então** devo receber uma mensagem automática de encerramento  
**E** a mensagem deve incluir o status final e orientações adicionais  

### Cenário 2 (negativo) – Falha no envio da mensagem
**Dado que** minha denúncia foi classificada como "Finalizado"  
**E** ocorre um erro no sistema de notificações  
**Quando** o sistema tenta enviar a mensagem de encerramento  
**Então** o sistema deve registrar a tentativa falha  
**E** tentar reenviar a mensagem após 1 hora
